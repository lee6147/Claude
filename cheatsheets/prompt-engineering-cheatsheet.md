# 프롬프트 엔지니어링 치트시트

> 개발자를 위한 AI 프롬프트 핵심 패턴 — 한 페이지 요약

## 기본 원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **구체적으로** | 모호한 표현 대신 정확한 지시 | ❌ "코드 개선해줘" → ✅ "이 함수의 시간 복잡도를 O(n²)에서 O(n)으로 줄여줘" |
| **컨텍스트 제공** | 기술 스택, 제약조건, 배경 정보 포함 | "Next.js 14 App Router, TypeScript strict 모드" |
| **출력 형식 지정** | 원하는 결과물의 형태를 명시 | "표 형식으로", "코드 블록으로", "단계별로" |
| **역할 부여** | AI에게 전문가 역할을 지정 | "시니어 백엔드 개발자로서 리뷰해줘" |
| **제약조건 명시** | 하지 말아야 할 것도 알려주기 | "외부 라이브러리 추가하지 말고 표준 라이브러리만 사용" |

## 코딩 프롬프트 패턴

### 1. 분석 우선 패턴

코드를 수정하기 전에 먼저 분석을 요청하는 패턴이에요. AI가 맥락을 이해한 후 더 정확한 수정을 제안합니다.

```
이 파일을 분석해줘. 수정하지 말고 분석만.
- 현재 아키텍처 구조
- 잠재적인 문제점
- 의존성 관계
```

### 2. 단계적 구현 패턴

복잡한 작업을 작은 단계로 나눠서 요청하는 패턴이에요.

```
[기능]을 구현하려고 해.

1단계: 데이터 모델 설계만 먼저 보여줘
2단계: 내가 확인한 후 API 엔드포인트 구현
3단계: 테스트 코드 작성
```

### 3. 제약 조건 패턴

원하지 않는 결과를 방지하기 위해 명시적으로 제한을 두는 패턴이에요.

```
인증 미들웨어를 구현해줘.

조건:
- JWT 기반, RS256 알고리즘
- 기존 User 모델 수정하지 않기
- 외부 라이브러리는 jose만 사용
- 에러 시 JSON 형식으로 응답
```

### 4. 예시 기반 패턴

원하는 출력 형태를 예시로 보여주면 AI가 패턴을 파악합니다.

```
아래 형식으로 API 응답 타입을 정의해줘:

// 예시
type UserResponse = {
  data: User;
  meta: { requestId: string; timestamp: number };
}

// 이 패턴으로 OrderResponse, ProductResponse도 만들어줘
```

### 5. 리뷰어 패턴

AI에게 코드 리뷰어 역할을 부여하는 패턴이에요.

```
이 PR을 시니어 개발자로서 리뷰해줘.

집중해서 봐야 할 점:
- 보안 취약점 (SQL injection, XSS)
- 성능 병목 가능성
- 에러 처리 누락
- 네이밍 컨벤션

각 이슈에 심각도(높음/중간/낮음) 표시해줘.
```

## 도구별 팁

### Claude Code

| 상황 | 프롬프트 팁 |
|------|-----------|
| 대규모 리팩토링 | `plan` 모드에서 계획 먼저 세우기 |
| 파일 탐색 | "이 프로젝트의 인증 관련 파일들을 찾아줘" |
| 디버깅 | "이 에러 메시지를 분석하고 원인 3가지를 추정해줘" |
| 테스트 | "이 함수의 엣지케이스를 포함한 테스트 5개 작성해줘" |

### GitHub Copilot

| 상황 | 프롬프트 팁 |
|------|-----------|
| 인라인 제안 | 함수 이름과 주석을 먼저 작성하면 더 정확한 제안 |
| Copilot Chat | `@workspace`로 프로젝트 전체 컨텍스트 참조 |
| 슬래시 명령 | `/explain`, `/fix`, `/tests` 활용 |
| Agent 모드 | "이 이슈를 해결해줘 #123" 형태로 작업 위임 |

### Cursor

| 상황 | 프롬프트 팁 |
|------|-----------|
| 코드 편집 | `Cmd+K`로 인라인 편집, 선택 영역 지정 후 지시 |
| 컨텍스트 추가 | `@파일명`으로 관련 파일 참조 |
| 규칙 설정 | `.cursorrules`에 프로젝트 컨벤션 정의 |
| 멀티 파일 | Composer에서 여러 파일 동시 수정 지시 |

## 흔한 실수 & 해결

| 실수 | 문제점 | 해결 |
|------|--------|------|
| "코드 짜줘" | 너무 모호함 | 언어, 프레임워크, 입출력 형식 명시 |
| 한번에 모든 것 요청 | 품질 저하 | 단계별로 나눠서 요청 |
| 결과를 그대로 사용 | 버그/보안 위험 | 반드시 리뷰 후 적용 |
| 컨텍스트 없이 질문 | 부정확한 답변 | 기술 스택, 버전, 제약조건 제공 |
| 동일 프롬프트 반복 | 같은 결과 반복 | 접근 방식을 바꿔서 재요청 |
| AI 출력을 맹신 | 잘못된 코드 배포 | 테스트 코드로 검증 |

## 고급 기법

### 메타 프롬프트

AI에게 프롬프트 자체를 개선하도록 요청하는 기법이에요.

```
내가 아래 프롬프트로 API를 구현하려고 하는데,
이 프롬프트를 더 구체적이고 효과적으로 다듬어줘:

"로그인 API 만들어줘"
```

### 체인 오브 싱킹 (Chain of Thought)

복잡한 문제에서 AI가 추론 과정을 단계별로 보여주도록 유도합니다.

```
이 알고리즘의 시간 복잡도를 분석해줘.
단계별로 추론 과정을 보여주고, 최종 결론을 내려줘.
```

### 셀프 검증 패턴

AI가 자신의 출력을 스스로 검증하도록 하는 패턴이에요.

```
위에서 작성한 코드에 대해:
1. 엣지케이스 3가지를 찾아줘
2. 보안 취약점이 있는지 확인해줘
3. 있다면 수정된 버전을 보여줘
```
