# 가이드 03: 일일 AI 코딩 루틴

> AI 코딩 도구를 매일 활용하는 실전 워크플로

## 아침 루틴 (5분)

```bash
# 1. 어제 작업 이어받기
claude "어제 작업 컨텍스트를 파악해줘. git log와 TODO를 확인하고 요약해줘"

# 2. 오늘 할 일 확인
claude "오늘 해야 할 작업을 우선순위대로 정리해줘"
```

## 코딩 중

### 새 기능 구현

```bash
# 1. 계획 먼저
claude "이 기능을 구현하려면 어떤 파일을 수정해야 하는지 분석해줘. 수정하지 말고 계획만."

# 2. 구현
claude "계획대로 구현해줘. 한 파일씩 진행하고 각 단계마다 테스트해줘."

# 3. 셀프 리뷰
claude "방금 변경한 코드를 리뷰해줘"
```

### 버그 수정

```bash
# 1. 원인 분석
claude "이 에러의 원인을 찾아줘: [에러 메시지]"

# 2. 수정
claude "근본 원인을 수정해줘. 최소한의 변경으로."

# 3. 회귀 방지
claude "이 버그가 재발하지 않도록 테스트를 추가해줘"
```

## 퇴근 전 (3분)

```bash
# 1. 정리
claude "오늘 작업한 내용을 커밋 메시지와 함께 정리해줘"

# 2. 내일 인수인계
claude "내일 이어서 해야 할 작업을 TODO로 남겨줘"
```

## 주간 루틴

```bash
# 코드 품질 체크 (금요일)
claude "이번 주에 추가된 코드의 품질을 분석해줘.
테스트 커버리지, 복잡도, 중복 코드 관점에서."
```

## 팁

- **컨텍스트 유지:** CLAUDE.md가 최신 상태인지 주기적으로 확인
- **패턴 축적:** 자주 쓰는 프롬프트는 alias로 등록
- **학습:** Claude의 제안 중 몰랐던 패턴은 메모
