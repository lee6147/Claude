# AI 코딩 치트시트

> AI로 10배 빠르게 빌드하기 위한 핵심 패턴 — 한 페이지 요약

## Claude Code 필수 명령어

| 명령어 | 용도 |
|--------|------|
| `claude` | 대화형 세션 시작 |
| `claude "prompt"` | 원샷 실행 |
| `claude -p "prompt"` | 파이프 모드 (스크립트용) |
| `/help` | 도움말 |
| `/clear` | 컨텍스트 초기화 |

## 효과적인 프롬프트 패턴

### 분석 요청 (수정 전)

```
[파일/코드]를 분석해줘. 수정하지 말고 분석만.
- 복잡도
- 중복 코드
- 개선 가능한 패턴
```

### 구현 요청

```
[기능]을 구현해줘.
조건:
1. [제약 조건 1]
2. [제약 조건 2]
3. 테스트도 함께 작성
```

### 리뷰 요청

```
현재 변경사항을 리뷰해줘.
체크: 보안, 에러핸들링, 타입안전성, 성능
```

### 디버깅 요청

```
이 에러를 해결해줘: [에러 메시지]
- 에러 로그: [로그]
- 재현 조건: [조건]
```

## CLAUDE.md 핵심 구조

```markdown
# 프로젝트명
## 기술 스택 — 무엇을 쓰는지
## 코딩 규칙 — 어떻게 작성하는지
## 금지 사항 — 무엇을 하면 안 되는지
## 명령어 — 빌드/테스트/배포 방법
```

**50-100줄이 최적.** 길면 무시됨.

## Git + AI 워크플로

```bash
# 리뷰 후 커밋
claude "변경사항 리뷰 후 문제 없으면 커밋 메시지 제안해줘"

# PR 설명 생성
git diff main...HEAD | claude "PR 설명을 작성해줘"

# 충돌 해결
claude "이 merge conflict를 해결해줘"
```

## 흔한 실수 & 해결

| 실수 | 해결 |
|------|------|
| "아무거나 해줘" | 구체적인 조건과 범위 제시 |
| 한 번에 너무 많이 | 파일 단위로 나눠서 요청 |
| CLAUDE.md 없이 시작 | 먼저 CLAUDE.md 작성 |
| AI 결과 무검증 커밋 | 항상 diff 확인 후 커밋 |
| 테스트 없이 리팩토링 | 테스트 먼저 → 리팩토링 |
