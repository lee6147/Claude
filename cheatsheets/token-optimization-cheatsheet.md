# 토큰 최적화 치트시트

> Claude Code 비용을 50% 이상 절감하는 설정과 습관 — 한 페이지 요약

## settings.json 핵심 설정

```json
{
  "model": "sonnet",
  "preferences": {
    "fastMode": true
  },
  "env": {
    "MAX_THINKING_TOKENS": "10000",
    "AUTOCOMPACT": "50",
    "SUBAGENT_MODEL": "haiku"
  }
}
```

| 키 | 값 | 효과 |
|----|------|------|
| `model` | `sonnet` | 기본 모델 — 대부분 작업에 충분해요 |
| `MAX_THINKING_TOKENS` | `10000` | 과도한 사고 토큰 방지 (기본값 무제한) |
| `AUTOCOMPACT` | `50` | 50% 컨텍스트 사용 시 자동 압축 |
| `SUBAGENT_MODEL` | `haiku` | 서브에이전트는 경량 모델로 비용 절감 |

## 모델 선택 가이드

| 작업 유형 | 추천 모델 | 이유 |
|----------|----------|------|
| 일반 코딩 | Sonnet | 속도-품질 균형 최적 |
| 복잡한 아키텍처 | Opus | 긴 사고가 필요한 작업 |
| 서브에이전트/검색 | Haiku | 단순 작업에 비용 최소화 |
| 빠른 반복 작업 | Sonnet + `fastMode` | 출력 속도 우선 |

> **팁:** Opus는 Sonnet 대비 ~5배 비용이에요. 정말 필요한 순간에만 사용하세요.

## 컨텍스트 관리 명령어

| 명령어 | 용도 |
|--------|------|
| `/compact` | 수동 컨텍스트 압축 — 대화가 길어질 때 |
| `/clear` | 컨텍스트 완전 초기화 |
| `/cost` | 현재 세션 토큰 사용량 확인 |
| `/model sonnet` | 모델 즉시 전환 |

## 비용 절감 5대 습관

### 1. CLAUDE.md를 잘 써요

```markdown
# CLAUDE.md
- 프로젝트 컨텍스트를 미리 기술
- AI가 매번 탐색하지 않아도 돼요
- 반복 질문 → 토큰 낭비 방지
```

### 2. 서브에이전트 모델을 분리해요

```json
{ "env": { "SUBAGENT_MODEL": "haiku" } }
```

Agent tool로 파일 탐색, 검색 같은 단순 작업을 위임할 때 Haiku면 충분해요.

### 3. 자동 압축을 켜요

`AUTOCOMPACT: 50`이면 컨텍스트 50% 도달 시 자동 압축해요. 수동 `/compact`를 잊어도 안전해요.

### 4. 프롬프트를 구체적으로 써요

```
❌ "이 코드 고쳐줘"
✅ "src/auth.ts의 validateToken 함수에서 만료 체크 누락 수정해줘"
```

모호한 프롬프트 → AI가 탐색에 토큰 소비 → 비용 증가.

### 5. MCP 서버를 10개 이하로 유지해요

활성 MCP 서버마다 스키마가 컨텍스트에 주입돼요. 사용하지 않는 서버는 비활성화하세요.
