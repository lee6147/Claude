# 14. AI 코딩 도구 비용 최적화 가이드

> AI 코딩 에이전트를 매일 쓰면서도 비용을 합리적으로 관리하는 실전 전략

## 왜 비용 최적화가 중요한가

AI 코딩 도구가 개발자의 필수 도구로 자리잡으면서, 월 구독료와 API 사용량이 무시할 수 없는 비용이 되었다. 특히 에이전틱 코딩 — AI가 자율적으로 여러 파일을 탐색하고 수정하는 워크플로우 — 에서는 토큰 사용량이 기하급수적으로 늘어난다.

2026년 3월 기준, 주요 AI 코딩 도구의 월 비용:

| 도구 | 플랜 | 월 비용 | 특징 |
|------|------|---------|------|
| Claude Code (Max) | Max 5 | $100 | Opus 무제한* |
| Claude Code (Max) | Max 20 | $200 | 높은 사용량 한도 |
| GitHub Copilot | Individual | $10 | 자동완성 + 채팅 |
| Cursor | Pro | $20 | 500 fast requests |
| Windsurf | Pro | $15 | Cascade 에이전트 |
| Codex CLI | API | 사용량 기반 | gpt-5.x 토큰 과금 |

> *"무제한"이라고 해도 rate limit은 존재한다. 실제 Anthropic의 Claude Code 사용자당 비용이 $5,000이라는 루머가 있었지만, [실제 분석](https://martinalderson.com/posts/no-it-doesnt-cost-anthropic-5k-per-claude-code-user/)에 따르면 과장된 수치다.

## 핵심 원칙: 토큰은 돈이다

### 1. 컨텍스트 윈도우 = 비용의 핵심

AI 코딩 도구의 비용은 **입력 토큰 × 출력 토큰**으로 결정된다. 컨텍스트에 불필요한 파일을 많이 넣으면 모든 요청의 비용이 증가한다.

```
❌ "전체 프로젝트를 분석해줘"
✅ "src/auth/login.ts의 validateUser 함수를 리뷰해줘"
```

**실전 팁:**
- `.claudeignore` / `.cursorignore`로 불필요한 파일 제외
- `node_modules/`, `dist/`, `*.lock` 등은 반드시 제외
- 대규모 JSON/CSV 데이터 파일 제외
- 빌드 아티팩트, 로그 파일 제외

### 2. 모델 선택은 태스크에 맞게

모든 작업에 최고급 모델을 쓸 필요 없다:

| 태스크 | 권장 모델 | 이유 |
|--------|----------|------|
| 자동완성 | 소형 모델 (Copilot) | 빈도 높고 단순 |
| 코드 리뷰 | 중형 모델 (Sonnet) | 패턴 매칭 중심 |
| 아키텍처 설계 | 대형 모델 (Opus) | 깊은 추론 필요 |
| 디버깅 | 대형 모델 (Opus) | 복잡한 인과 추론 |
| 테스트 생성 | 중형 모델 (Sonnet) | 패턴 기반 생성 |
| 문서 생성 | 중형 모델 (Sonnet) | 구조화된 출력 |

**Claude Code에서 모델 전환:**
```bash
# 세션 중 모델 변경
/model sonnet    # 가벼운 작업
/model opus      # 복잡한 작업
```

### 3. 프롬프트 효율화

짧고 명확한 프롬프트가 비용도 적고 결과도 좋다:

```
❌ "이 코드에 문제가 있는 것 같은데 뭔지 모르겠어. 전체적으로 한번 봐주고 
    개선점도 알려주고, 테스트도 만들어주고, 문서도 업데이트해줘"

✅ "validateUser()에서 null 체크가 빠져있어. 수정하고 엣지케이스 테스트 추가해줘"
```

## 구독 vs API: 어떤 모델이 유리한가

### 구독형이 유리한 경우
- 하루 3시간 이상 코딩에 AI 사용
- 주로 대형 모델(Opus급) 필요
- 예측 가능한 월 비용 선호

### API 과금이 유리한 경우
- 간헐적 사용 (주 2-3회)
- 자동화 파이프라인 (CI/CD, 크론)
- 소형 모델 위주 사용

### 비용 시뮬레이션 예시

```
시나리오: 하루 2시간 코딩, Opus 모델 사용

구독형 (Claude Max 5): $100/월 고정
API형: 약 $50-150/월 (사용 패턴에 따라 변동)

→ 꾸준히 쓰면 구독형, 가끔 쓰면 API가 유리
```

## 실전 비용 절감 전략

### 전략 1: 컨텍스트 프리로딩

매 요청마다 컨텍스트를 새로 보내지 말고, 세션을 유지하라:

```bash
# Claude Code: 하나의 세션에서 연속 작업
# (세션 유지 = 캐시 히트 = 비용 절감)
claude --resume

# Cursor: Composer에서 대화 이어가기
# 새 채팅 열지 말고 기존 컨텍스트 활용
```

### 전략 2: CLAUDE.md 최적화

프로젝트 규칙을 `CLAUDE.md`에 정리하면 매번 설명할 필요 없다:

```markdown
# CLAUDE.md
- TypeScript strict mode 사용
- 테스트: vitest, 커버리지 80% 이상
- 컨벤션: kebab-case 파일명, barrel export
- DB: Prisma ORM, PostgreSQL
```

**이점:** 반복 프롬프트 제거 → 입력 토큰 절감

### 전략 3: 2단계 워크플로우

1. **Plan 단계** (소형 모델): 무엇을 할지 계획
2. **Execute 단계** (대형 모델): 계획대로 실행

```bash
# Step 1: Sonnet으로 계획
/model sonnet
"이 리팩토링 계획을 세워줘. 코드 작성하지 말고."

# Step 2: Opus로 실행
/model opus
"위 계획대로 실행해줘."
```

### 전략 4: 캐시 활용

Claude API의 프롬프트 캐시를 활용하면 반복 컨텍스트 비용을 90% 절감:

- 시스템 프롬프트와 공통 파일은 캐시 대상
- 5분 이내 재요청 시 캐시 히트
- Claude Code는 자동으로 캐시 최적화

### 전략 5: 도구 조합 전략

모든 것을 하나의 도구로 해결하지 마라:

```
자동완성        → Copilot ($10/월)
에이전틱 코딩    → Claude Code ($100/월)
빠른 질문       → ChatGPT/Claude.ai (무료)
CI/CD 자동화    → Codex CLI (API, 저비용)
```

총 비용: ~$110/월로 전 영역 커버

## 비용 모니터링

### Claude Code 사용량 확인

```bash
# 세션 비용 확인
/cost

# Anthropic 대시보드
# console.anthropic.com → Usage
```

### 월간 체크리스트

- [ ] 구독 플랜 사용량 대비 비용 확인
- [ ] 불필요한 구독 정리 (사용하지 않는 도구)
- [ ] `.claudeignore` 업데이트
- [ ] 팀원 간 비용 비교 → 베스트 프랙티스 공유

## 팀 환경에서의 비용 관리

### API 키 관리
```bash
# 팀원별 API 키 분리
ANTHROPIC_API_KEY_DEV=sk-ant-...
ANTHROPIC_API_KEY_CI=sk-ant-...

# 사용량 제한 설정 (Anthropic Console)
# - 일일 한도
# - 월간 한도
# - 모델별 제한
```

### 비용 할당
- 프로젝트별 API 키 분리
- 월간 비용 리포트 자동화
- 비용 이상 알림 설정

## 요약: 80/20 법칙 적용

비용 최적화의 80%는 이 3가지로 달성된다:

1. **컨텍스트 최소화** — 필요한 파일만 포함
2. **모델 매칭** — 태스크 복잡도에 맞는 모델 선택
3. **세션 유지** — 캐시 활용으로 반복 비용 제거

> 💡 핵심: AI 코딩 도구에 돈을 아끼는 것보다, **AI로 절약하는 시간의 가치**가 훨씬 크다. 최적화는 하되, 생산성을 희생하지 마라.
